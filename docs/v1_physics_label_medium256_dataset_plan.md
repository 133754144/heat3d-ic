# Heat3D v1 Physics-Label Medium256 Dataset Plan

## Purpose

`medium256` moves the current 64-sample physics-label medium smoke toward a
larger benchmark-candidate dataset preparation step. It is intended to exercise
the same generation, checking, training-export, and comparison interfaces at a
larger scale while keeping the same non-claim boundary.

This is medium256 dataset tooling / generation smoke / benchmark-candidate
dataset preparation. It is not a formal benchmark, not high-fidelity solver
evidence, not model-performance evidence, and not OOD generalization evidence.

## Manifest

Manifest:

```text
configs/heat3d_v1_physics_label_medium256_manifest.json
```

Planned generated subset:

```text
data/heat3d-thermal-simulation/subsets/v1_multilayer_bc_eq_physics_label_medium256_v2
```

The manifest records a deterministic combination seed:

```text
20260511
```

The sample combinations are generated by deterministic balanced cycling, not by
runtime-random selection.

## Coverage Dimensions

The manifest covers the current medium_v2 design space:

- source patterns: `centered_single_hotspot`, `shifted_single_hotspot`,
  `edge_or_corner_hotspot`, `two_hotspots_same_layer`,
  `dual_active_layer_hotspots`, `broad_block_power`, `multi_block_power`
- k region modes: `layerwise_isotropic_k`, `blockwise_isotropic_k`,
  `interposer_equivalent_k`, `diagonal_anisotropic_k`
- k field modes: `iso1`, `diag3`
- stack templates: `baseline_4_layer`, `compact_3_layer`,
  `interposer_like_4_layer`, `dual_active_4_layer`,
  `held_out_interposer_like_candidate`
- BC categories: `nominal_top_h`, `low_top_h`, `high_top_h`,
  `held_out_top_h_candidate`

## Generation Strategy

The generator reuses the current medium generation path:

- region-first sample construction
- rectilinear grid sampling
- `volume_fraction` source assignment
- `fixed_density` q policy
- `heat3d_v1_reference_solver_v2` labels

Each generated sample writes:

- `coords.npy`
- `k_field.npy`
- `q_field.npy`
- `temperature.npy`
- `sample_meta.json`
- `label_meta.json`

## Local Smoke Policy

Mac local validation should generate only a partial smoke subset:

```bash
python3 tools/generate_heat3d_v1_physics_label_medium.py \
  --manifest configs/heat3d_v1_physics_label_medium256_manifest.json \
  --output-subset data/heat3d-thermal-simulation/subsets/v1_multilayer_bc_eq_physics_label_medium256_local_smoke_v2 \
  --sample-limit 16 \
  --write \
  --overwrite
```

The first 16 manifest samples include train and valid samples so the controlled
training export runner can perform a tiny acceptance smoke if needed. This
local smoke is not a complete medium256 generation.

## Full Run Location

Full 256-sample generation and longer controlled training should run in the SSH
Git-only directory:

```text
~/myCodeGitOnly/heat3d-ic
```

Generated `data/`, `output/`, checkpoint, and log artifacts must stay ignored
unless a later explicit artifact policy is approved.

## Non-Claims

Do not describe medium256 outputs as:

- a formal benchmark
- high-fidelity solver validation
- model performance
- OOD generalization
- industrial 3D IC thermal simulation validity
