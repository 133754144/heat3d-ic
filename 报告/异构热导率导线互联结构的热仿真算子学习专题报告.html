异构热导率导线互联结构的热仿真算子学习专题报告

基于本地论文存档与已保存联网结果整理
关注问题：在热仿真的算子学习框架中，如何处理含不同热导率的导线互联结构
完成时间：2026年4月14日

术语说明：本报告中的“导线互联结构”主要包括 TSV / TTSV、bump / micro-bump、金属互连与周围介质复合层，以及 chiplet 层间插层等结构。严格来说，当前 3D IC 热仿真 AI 文献更多聚焦 TSV、bump 和插层，而对更细粒度 BEOL 导线网络的显式算子学习研究仍较少。

核心判断：现有算子学习文献处理“不同热导率互联结构”的主流方法，不是直接逐根导线显式求解，而是围绕“导热率场编码、等效热导率提取、界面物理约束、几何感知表示、多保真与混合求解”五条路线展开。

一、问题背景：为什么异构热导率互联结构对算子学习特别困难

在 3D IC / chiplet 热分析中，互联结构往往同时包含高导热金属和低导热介质。例如，TSV 及 bump 的金属核心与 Si、SiO2、underfill、bonding layer 等材料之间存在显著热导率反差；而层间互联阵列又会在局部形成极强的热扩散通道或热阻瓶颈。对算子学习而言，这种问题并不只是“输入材料参数不同”那么简单，而是同时带来了热导率跳变、多尺度几何、界面连续性约束和结构变化四类困难。

第一，互联结构会引入强烈的分段常数或高对比度导热率场。DeepOHeat 已经把 full-chip flexible material conductivity distribution 视为可以与功率图、边界条件一起输入的 PDE 配置之一 [1]，这说明作者已经意识到导热率分布本身就是需要被学习的“函数输入”，而不仅是静态背景。第二，TSV 和 bump 通常包含超薄绝缘层和细尺度金属几何，PI-ONet 明确指出这些结构会带来严重的网格退化和多尺度建模复杂性 [2]。第三，材料交界面上的温度连续与热流连续条件若处理不好，模型即使平均误差不高，也容易在局部热点和界面附近出现不可信结果 [2]。第四，互联结构还常常随着阵列密度、尺寸比例、排布方式或界面缺陷而变化，外部工作中的 Adaptive Graph Learning 已经证明这类“界面变化”本身就是一个需要专门处理的问题 [3]。

难点1：热导率强跳变

高导热金属与低导热介质并存，会导致局部热流重新分配和温度梯度突变。

难点2：多尺度几何

细导线、超薄氧化层和大尺度芯片区域共同存在，直接精细离散代价极高。

难点3：界面条件

需要同时满足温度连续与热流连续，尤其在异质材料层间最敏感。

难点4：结构变化

互联尺寸、阵列密度、界面缺陷和封装形态变化会破坏单一模板模型的泛化能力。

二、现有算子学习及相关文献的五类处理策略

2.1 把导热率分布直接作为输入函数编码

这是最直接的策略：不先做等效化，而是把导热率场本身作为算子学习输入的一部分。DeepOHeat 的论文贡献中明确写到，其模块化编码方式可以处理 “2D/3D power maps、boundary conditions，以及 full-chip flexible material conductivity distribution” [1]。这说明在原则上，算子学习可以把空间导热率分布和功率图、边界条件一样视作函数输入，通过 branch/trunk 结构学习材料场到温度场的映射。

这种策略的优点是表达直接、物理语义清楚，而且不需要在建模前就决定哪些细结构必须被均匀化。其局限也很明显：当互联结构非常细、材料跳变很剧烈、几何高度不规则时，简单的规则网格表示会迅速推高分辨率需求，训练和推理都可能变得昂贵。因此，单纯“把导热率场塞进网络”通常只适合尺度尚可解析、几何变化不太剧烈或已经有统一网格表示的场景。

2.2 先提取等效热导率，再做大尺度算子学习

从目前本地文献看，这是处理互联结构最现实、也最常见的路线。PI-ONet 将 TSV 和 bump 层视为最突出的热建模难点之一，并专门设计了热提取模块：先对单个 TSV 或 bump building block 用 FEM 提取等效导热率，再训练一个 ANN 根据几何参数预测其水平和垂直等效热导率，最后把整个 TSV/bump 层当作具有等效导热率的均匀材料纳入更大尺度热分析 [2]。其好处是显著减少了需要显式处理的细粒度几何复杂度。

与此相对应，Fast Thermal Modeling of TTSV via Bounded Neural Networks 更进一步聚焦于 TTSV 的各向异性等效热导率提取。该工作把解析 Series-Parallel Thermal Conductivity 模型与带物理边界的 BPNN 结合，用有界输出层预测最难的水平等效热导率，并把误差控制在 FEM 基准的 0.35% 以内 [4]。这类方法的意义在于：当导线互联结构过于精细，直接做显式算子学习不划算时，可以先把它们压缩成“局部各向异性热导率张量”或等效层参数，再交给上层算子学习框架。

从专题问题的角度看，这条路线实际上回答了一个非常关键的问题：当互联结构本身比全芯片热场的解析尺度更细时，先做局部等效参数提取，再做全局算子学习，是现阶段最可行的分层策略。

2.3 在算子学习中显式加入界面条件

仅有等效导热率还不够，异质材料接触时最敏感的问题之一是界面处理。PI-ONet 在这方面给出了很直接的做法：对于 multilayer chiplet，它在物理损失中显式写入层间界面的温度连续和热流连续条件，并在网格化实现时分别在界面上下两侧的离散点上评估这些条件 [2]。换句话说，这不是把“界面影响”留给网络自己去猜，而是明确告诉模型：在材料分界面上，哪些物理关系必须满足。

对于含不同热导率互联结构的算子学习，这一点非常重要。因为如果只输入等效参数而没有界面约束，模型很容易在材料跳变附近学出“看起来平均误差不高、但局部热流不对”的温度场。PI-ONet 的混合物理+数据训练说明，在互联结构与异质层交界处，算子学习若想真正可用，最好显式嵌入界面连续性约束，而不是完全依赖纯数据拟合。

2.4 用几何感知或图算子处理不规则互联结构

当互联几何不再适合简单地投影到规则网格时，几何感知算子和图算子就变得有吸引力。尽管 GINO 不是芯片热仿真论文，但它从方法学上给出了一个非常有价值的方向：用点云和 SDF 表示变化几何，用图神经算子在不规则域上做局部积分，再映射到规则潜空间里用 FNO 处理全局耦合，最终再投回原始几何域 [5]。这种做法本质上是在回答“复杂几何如何进入神经算子”。

在芯片热分析场景中，外部工作的 Adaptive Graph Learning for Efficient Thermal Analysis of Multi-Stacking Chiplet Systems under Interface Variations 已经显示出这条路线的现实意义。该工作面向 2.5D/3D multi-stacking chiplet systems，用 GNN + FEM hybrid 处理界面变化，并结合真实 post-silicon 热成像验证鲁棒性 [3]。虽然它不是纯算子学习论文，但它表明：一旦互联结构和界面变化成为问题核心，图表示和几何感知能力很可能比单纯规则网格上的高精度更重要。

2.5 用多保真与混合求解缓解精细互联带来的数据与可信度问题

导线互联结构越细、材料差异越大，显式高保真仿真越贵，这会直接推高算子学习所需标签成本。因此，多保真与混合求解就不仅仅是优化技巧，而是解决互联结构问题的重要辅助手段。ARO 通过多保真融合和主动学习减少高保真样本需求 [6]；T-Fusion 在瞬态预测中只用很少的高精度数据就能保持 1 K 内误差 [7]；DeepOHeat-v1 进一步通过 confidence score 与 FD/GMRES 协同修正预测结果 [8]。

这些工作共同指向一个结论：对于包含异构互联结构的热仿真问题，最现实的高质量路线不是“纯神经网络一次到位”，而是“局部等效参数/低保真模型 + 少量显式高保真样本 + 算子学习代理 + 必要时的局部数值修正”的混合框架。

四、现有文献对“导线互联结构”处理到什么程度了

结合本地存档和已检索资料，可以得到一个相对清晰的判断：现有 3D IC 热仿真算子学习论文，对 TSV、bump、插层和层间界面已经开始有比较明确的处理方案，但对更细粒度的 BEOL 金属线/介质网络的显式算子学习仍然比较少。更具体地说，当前研究大多落在以下两个层次。

第一层是“把互联结构当作必须参数化和等效化的微结构”，这是 PI-ONet 与 TTSV 等效热导率建模工作的核心思路 [2][4]。第二层是“把互联和界面变化当作更复杂几何与图结构问题的一部分”，这在 GINO 和 Adaptive Graph Learning 的方法逻辑中能够看到未来方向 [3][5]。但无论哪一层，目前都还缺少一套成熟的、专门针对“显式不同热导率导线互联网络”的热算子学习标准框架。

综合判断：如果你的目标是研究“含不同热导率导线互联结构”的算子学习，当前最合理的定位不是宣称已有方法已经解决了这个问题，而是指出：现有工作已经给出了若干关键组件，但尚未把“异构导热率微结构 + 任意几何 + 物理界面 + 可迁移算子学习”真正统一起来。

五、对后续研究的具体建议

5.1 建议采用三级建模思路，而不是一开始就端到端显式求全

对于含有不同热导率导线互联结构的问题，我建议把后续研究路线设计为三级框架。第一级是局部互联单元的等效参数学习：以 TSV / bump / 局部互联单元为对象，学习其几何参数到各向异性等效热导率张量的映射，这一层可以直接借鉴 PI-ONet 和 TTSV bounded NN 的思路 [2][4]。第二级是全局热场的几何感知算子学习：将上一级得到的等效导热率场、功率图、边界条件和界面掩码作为输入，用图神经算子或任意域神经算子预测全局温度场。第三级是可信度评估和局部数值修正：对互联密集区、热点区或不确定性高的区域，调用高保真求解器做增量修正，这一层可以借鉴 DeepOHeat-v1 的混合求解思路 [8]。

5.2 输入不应只有功率图，还应显式包含“导热率张量场 + 界面标识”

如果研究目标是异构互联结构，那么仅输入功率图显然不够。更合理的输入应至少包含四类信息：功率场、材料导热率场或等效导热率张量场、边界条件，以及界面或材料类别标识。DeepOHeat 已经证明导热率分布本身可以作为函数输入 [1]；PI-ONet 又说明界面条件可以作为物理损失显式加入 [2]。将这两者结合起来，是比单纯堆叠网络更有研究价值的方向。

5.3 若显式几何很复杂，图表示可能比规则网格更合适

对于规则化、层级化的 3D IC 场景，规则网格仍然方便；但如果你后续真的想把互联结构做得更细、更不规则，图表示或点云/SDF 表示很可能更自然。GINO 的方法学价值在于说明了“复杂几何并不一定必须先规则化成体素才能进入神经算子” [5]。如果你的目标包含任意布局、局部互联密度变化或界面缺陷，这条路线会比单纯 FNO 更有潜力。

5.4 评价指标必须加入局部热点与界面区域指标

对于异构导热率互联结构，平均温度场误差只是最基础的评价。更有意义的指标至少还应包括：峰值温度误差、互联密集区局部温度梯度误差、界面附近热流误差、热点定位偏差，以及在未见互联排布和未见材料参数下的泛化误差。否则，即使整体误差不高，也无法说明模型真的学会了互联结构对热扩散路径的影响。

六、专题结论

综上，当前热仿真算子学习领域处理“不同热导率导线互联结构”的方式，主要不是直接显式解析每一根细导线，而是通过五种策略协同完成：其一，把导热率分布直接作为函数输入；其二，对 TSV / bump / 插层等细结构先做等效热导率提取；其三，在训练中显式加入界面连续条件；其四，利用图算子或几何感知算子处理更复杂的互联几何；其五，通过多保真与混合求解框架降低精细结构带来的数据成本和可信度风险。

从研究空白看，当前最值得做的不是简单复现已有 3D IC 热算子学习论文，而是把“等效互联参数学习、几何感知算子、界面物理约束和局部高保真修正”统一到一个更适合异构互联结构的框架中。对你的选题而言，这恰好是一个很有辨识度、也与现有本地工作积累高度衔接的切入点。
